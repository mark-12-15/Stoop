# StoopKeep 产品策略

**产品定位**: 专注维修收据管理的房东工具  
**Slogan**: Keep every repair. Keep every receipt.

---

## 1. 市场竞品分析

### 主要竞品

#### Avail (Apartments.com)
- **维修功能**: 基础工单看板
- **租客体验**: ❌ 必须登录 Web Portal
- **税务功能**: ❌ 无 Schedule E 专项导出
- **用户抱怨**: "租客根本不用，还是发短信"

#### TurboTenant
- **维修功能**: 较完善，有状态流转
- **租客体验**: ❌ 需要下载 App 或登录
- **税务功能**: ❌ 只有通用记账
- **用户抱怨**: "功能太重，客服太慢"

#### Stessa (最大威胁)
- **维修功能**: 2024年新增，但锁在付费版
- **租客体验**: ❌ 强制登录租客门户
- **税务功能**: ✅ Schedule E 一键导出（核心优势）
- **定价**: $28/月 = $336/年（2024涨价）
- **用户抱怨**: "涨价太狠，维修功能是凑数的"

### 竞品盲区
1. **租客体验差**: 都需要登录/下载App，租客不愿用
2. **维修与财务脱节**: 维修工单和税务报表是两个系统
3. **价格贵**: Stessa $336/年，Avail Plus $1,080/年（10套房）

## 2. 核心痛点

### 真实用户痛点（Reddit/BiggerPockets）

> "租客发短信报修 → 我找人修 → Zelle 付钱 → 师傅给纸质收据。年底报税时，这三样东西散落在三个地方，对账是地狱。"

**高频抱怨关键词**:
- 📱 "找不到收据" (receipt hunting)
- 💸 "分不清 Repairs vs Improvements" (税务分类困惑)
- 📊 "Excel + 手机相册，乱得要死"
- 🕐 "报税季要花几天整理CPA要的材料"

### 目标用户（ICP）

**精准客户**: 5-15套房的"痛苦夹心层"小房东

**特征**:
- 自己管理房产（没雇物业经理）
- 年维修次数：20-50次
- Excel已经记不过来
- 但又舍不得花$200+/月请物业公司

**❌ 非目标用户**:
- 1-4套房业余房东（用Excel够了）
- 20+套房专业房东（已用Buildium/AppFolio）

## 3. 我们的差异化优势

### 产品差异化

1. **租客体验碾压**: 免登录链接提交（30秒完成）
   - 租客：收到短信链接 → 拍照描述 → 提交
   - 无需注册、下载App、记密码

2. **专注财务闭环**: 维修工单 → Schedule E报表
   - 维修记录自动分类（Repairs vs Improvements）
   - 一键导出IRS标准格式
   - CPA直接使用，无需二次整理

3. **AI照片分析**: Gemini Flash 1.5
   - 租客拍照 → AI识别问题类型
   - 收据照片 → 自动提取金额/供应商

4. **支持所有费用类型**: 
   - 不只是维修：保险、房产税、水电费等13种费用
   - 完整的Schedule E导出

### 技术差异化

- **Gemini Flash 1.5**: 比GPT-4 Vision成本低，速度快
- **免登录提交**: 基于property slug的公开链接
- **多源数据汇总**: 租客提交 + 手动导入 + CSV批量导入

## 4. 定价策略

### 当前定价（配额模型：3+3+3）

```
Free Plan（永久试用）:
├─ 租客提交工单：3个
├─ 房产数量：3个
├─ 导出工单：每次3个 ← 变现核心
└─ 手动导入：无限 ← 数据粘性

Pro Plan（$12/月或$99/年）:
├─ 无限租客提交
├─ 无限房产
├─ 无限导出
└─ 无限手动导入
```

### 定价逻辑

**核心原则**: 
```
输入端：适度开放（租客3个，手动无限）
输出端：严格限制（导出3个）
变现逻辑：数据进得去，出不来 → 要出来，付费
```

**用户旅程**：
1. 新用户导入历史数据（100个工单，免费）
2. 全年使用，积累更多数据（免费）
3. 报税季想导出 → 只能导出3个工单
4. 看到损失：$12,000税务抵扣 vs $300年费
5. **付费转化**

### 为什么手动导入不限制？

- ✅ 成本几乎为零（只是INSERT）
- ✅ 数据越多，迁移成本越高，粘性越强
- ✅ 即使限制，用户也可以删除旧工单再导入（防不住）
- ✅ 导出是唯一变现点，限制这里就足够

## 5. 定价竞争力分析

### 价格对比

| 产品 | 月付 | 年付 | 年度成本 |
|------|------|------|---------|
| **StoopKeep** | $12 | $99 | **$99** |
| Stessa Pro | $28 | - | $336 |
| TurboTenant | $16.5 | - | $198 |
| Avail Plus | $9/房产 | - | $1,080 (10套房) |

### 为什么$30/月有竞争力？

**1. 税务抵扣**: 
- 软件费用100%可抵税（Schedule E Line 8）
- 实际成本 = $300 × (1 - 税率22%) = $234/年

**2. 对比其他支出**:
- 物业保险: $1,000-2,000/年
- HOA费用: $200-500/月
- 维修工平均上门: $150-300/次
- CPA整理收据: $50-150/小时

**3. ROI计算**:
- 如果节省CPA 2小时时间 = $100-300
- 如果找回遗漏的维修抵扣 = $500-2,000
- 投资回报率: 200%-700%

**4. 心理定价**: 
- $30/月 = 每天$1 (低于一杯咖啡)
- $300/年 = 一次维修工上门费
- 比Stessa便宜（$336 → $300）

### 话术

> "只需$300/年（可全额抵税），帮你找回漏掉的几千刀维修抵税额，并省下CPA按小时收费的整理费。"

## 7. 风险与应对

### 风险1: Stessa降价或优化维修功能
**应对**: 我们的租客体验（免登录）是核心护城河

### 风险2: 用户认为"Excel够用了"
**应对**: 强调税务合规性和CPA友好性（Schedule E格式）

### 风险3: 获客成本高
**应对**: Reddit/Facebook房东社区有机增长，口碑传播

### 风险4: AI一致性问题（Gemini Flash）
**应对**: 
- 结构化输出（JSON Schema）
- 自动重试3次
- 关键信息需房东确认
